1.1.         FUNCTIONAL DETAILS (MANDATORY)

 

1.1.1. Detailed functional Description

This document is to re-define the new process to edit the attributes of an identity cube in the myid system. An identity cube is a collection of identity data that is used to manage access rights and permissions for different applications and systems. The identity attributes are the properties of an identity cube, such as name, email, manager, etc.

In the current process, GUA staffs have access to edit the identity attributes directly from the identity warehouse, which is the central repository of identity data. However, this direct update has created many data quality issues, mainly because the attribute updates in this area are free text fields and no validations are performed. This can lead to inconsistent, inaccurate, or incomplete identity data, which can affect the security and functionality of the myid system and the applications that rely on it.

To address these issues, we propose a new process that follows a proper validation and audit procedure to update the identity attributes. This document will explain the new process and the steps involved in editing the identity attributes.

1.1.1.1. Target Systems

 

The targets used in this case is myID.

 

1.1.1.2. Business Functionality

The new process to edit the identity attributes is as follows:

1. We will introduce a new quick link in the myid system that is accessible only to GUA staff.

2. We will disable the capability associated with editing the identity attributes in the identity warehouse.

3. The quick link will launch a workflow to update the identity attributes. The workflow will contain validations and quality checks to handle each of the fields that are updated.

4. Once the field level validations are passed, a confirmation screen will display the changes to be submitted.

5. Custom audit events will capture the changes and the source (requester) in detail.

6. Post audit, a provisioning plan with an access request (auto approved) will be generated to track the changes.

 

 

 

1.1.1.3. In Scope Requirements

The scope of this requirement is to introduce new quicklink form for GUA team to perform update identity attributes like as-is in the same way that they are doing through OOTB edit attributes screen.

As phase1 we will introduce validations only for the following fields :

1.1.2.     High Level Technical Information

The steps to update the identity attributes using the new process are as follows:

1. Log in to the myid system and click on the quick link that says "Edit Identity Attributes". This will launch the workflow to update the identity attributes.

2. Select the identity cube that needs to be updated.

3. Fill in the fields that need to be updated. The workflow will perform validations and quality checks on each field. For example, it will check if the email format is valid, if the manager exists in the system, if the identity type is consistent with the identity sub type, etc. If there are any errors or warnings, the workflow will display them and ask you to correct them or confirm them.

4. Once all the fields are filled in and validated, click on the "Submit" button. A confirmation screen will show the changes that will be made to the identity cube. Review the changes and click on the "Confirm" button.

5. The workflow will generate custom audit events that will capture the changes and the source (requester) in detail. These audit events will be stored in the myID system and can be viewed by authorized users.

6. The workflow will also generate a provisioning plan with an access request that will track the changes. The access request will be auto approved, and the changes will be applied to the identity cube.

7. The identity which is updated (owner in case it is secondary) , GUA team member will send notification that the identity attributes have been updated successfully.
