1.   Functional Design

1.1.         Functional Details (MANDATORY) 

  1. Detailed functional Description

The objective of this enhancement is to onboard the eBank Portal application which will support the following operations:

  1. Create User
  2. Enable user
  3. Disable the user
  4. Account aggregation
  5. Group Aggregation
  6. Adding single role to the user
  7. Adding the multiple roles to the user
  8. Removing the roles for the user
  9. Delete the user
  10. Send email notification on account creation

 

  1. Target Systems

JDBC Table in IIQ database

  1. Business Functionality

The application team needs the user’s data to create their Bank portal profiles. They will get the user details from JDBC table and with that they will create the profile in portal. This will be a disconnected application.

 

  1. In Scope Requirements

1.            Create User

2.            Enable user

3.            Disable the user

4.            Account aggregation

5.            Group Aggregation

6.            Adding/Removing single role to the user

7.            Adding the multiple roles to the user

8.            Removing the roles for the user

9.            Delete the user

10.         Send email notification on account creation

 

1.1.2.     High Level Technical Information

We will create JDBC provisional rules for Create, Disable, Enable, Delete and Modify functionalities.

There will be 2 tables in the database one for Users and one for Roles.
